# 📋 دستور مشروع "الولاء" — نظام ولاء الكافيهات (SaaS)

> هذا الملف مرجعنا الثابت. أي قرار معماري يُوثّق هنا.

---

## 🎯 الفكرة باختصار

نظام ولاء SaaS للكافيهات، بديل مبتكر لأنظمة الباركود المكلفة. كل كافيه (مستأجر/tenant)
يحصل على نظام ولاء كامل بهويته الخاصة، فيه:
- **روليت مكافآت** تُفتح فقط بعد عملية شراء فعلية (منع الاستغلال).
- **محفظة ولاء** للعميل (كم كوب جمّع / كم باقي على الكوب المجاني).
- **شاشة كاشير** بسيطة جداً للموظف.
- **لوحة تاجر** فيها إعدادات الهوية + الولاء + الروليت + تقارير تحليلية.

---

## 🏗️ المعمارية النهائية

```
المستخدمون (3 أنواع)
  ├── العميل    → صفحة محفظة الولاء (PWA)         [بدون تسجيل دخول — يدخل عبر رقم جواله]
  ├── الموظف    → شاشة الكاشير                     [إيميل + كلمة مرور]
  └── التاجر    → لوحة الإعدادات + التقارير          [إيميل + كلمة مرور]
                          ↓
              Next.js على Vercel
        (الواجهات + PWA + نشر تلقائي من GitHub)
                          ↓
                    Supabase
        ├── Postgres (البيانات + RLS متعدد المستأجرين)
        ├── Edge Functions (المنطق الحسّاس + أسرار واتساب)
        └── Auth (دخول التاجر والموظف بالإيميل/كلمة المرور)
```

---

## ✅ القرارات المحسومة

| القرار | الاختيار | ملاحظة |
|--------|----------|--------|
| الفرونت اند | **Next.js** | كامل (SSR متاح لو احتجناه) |
| الاستضافة | **Vercel** | مجاني، نشر تلقائي من GitHub |
| الباك اند + DB | **Supabase** | Postgres + Edge Functions + Auth |
| المنطق الحسّاس + الأسرار | **Supabase Edge Functions** | لا تلمس المتصفح |
| وصول كل كافيه لصفحته | **دومين فرعي** `cafe.site.com` | multi-tenant عبر subdomain |
| دخول الموظف/التاجر | **إيميل + كلمة مرور** | عبر Supabase Auth |
| دخول العميل | **بدون تسجيل** — عبر رقم الجوال | الموظف يُدخل الرقم |
| التحقق واتساب (مرحلة 1) | **زر يفتح `wa.me` والموظف يرسل يدوياً** | قابل للتطوير لـ Meta API |
| التحقق واتساب (مرحلة 2) | **Meta WhatsApp Cloud API** عبر Edge Function | مستقبلاً |

## 🔐 توزيع الأسرار (مهم جداً)

| السر | مكانه | السبب |
|------|-------|-------|
| `NEXT_PUBLIC_SUPABASE_URL` | Vercel → Env Vars | عام، آمن (بادئة `NEXT_PUBLIC_`) |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Vercel → Env Vars | عام، محمي بـ RLS |
| `SUPABASE_SERVICE_ROLE_KEY` | **Supabase Edge Secrets فقط** | خطير — صلاحيات كاملة |
| توكن واتساب (Meta) | **Supabase Edge Secrets فقط** | خطير — مرحلة 2 |

> ⚠️ لن نستخدم GitHub Actions Secrets — Vercel هو من ينشر.

## 🗺️ خطة العمل (مراحل)

- [ ] **م1 — الأساس:** Next.js + ربط Vercel + GitHub (نتأكد ينشر).
- [ ] **م2 — قاعدة البيانات:** جداول Supabase + RLS متعدد المستأجرين.
- [ ] **م3 — الموظف:** شاشة الكاشير + زر واتساب `wa.me`.
- [ ] **م4 — العميل:** محفظة الولاء + PWA + الروليت.
- [ ] **م5 — التاجر:** إعدادات الهوية (ألوان/شعار) + الولاء + الروليت.
- [ ] **م6 — التقارير:** التحليلات + التقسيمات الزمنية.
- [ ] **م7 — التطوير:** ربط Meta API + الإشعارات (Web Push).

## 🎨 إعدادات التاجر القابلة للتخصيص
**الهوية:** اسم الكافيه · الشعار · لون أساسي · لون ثانوي · لون خلفية · لون أزرار ·
نمط الواجهة (فاخر/بسيط/شبابي/كلاسيكي) · صورة خلفية · أيقونة التطبيق.
**الولاء:** عدد الأكواب للكوب المجاني · احتساب كوب أم فاتورة · حد يومي · اعتماد الموظف للكوب المجاني · مدة صلاحية المكافآت.
**الروليت:** لكل خانة → اسم الجائزة + نسبة الظهور + اللون.
